# Renderer v0.6 — Directional Shadow Maps — Design Spec

**Date:** 2026-07-01
**Status:** Approved for planning
**Scope:** Engine v0.6 — a directional-light shadow map: a depth-only shadow pass
from the light's POV, a Light uniform block, and PCF shadow sampling in the mesh
shader, with a ground-plane receiver so the swarm casts visible shadows.
**Builds on:** v0.5 render passes + render-to-texture (merged to `main`).

## 1. Goal & Motivation

The mesh shader lights the scene with a fixed directional light and no shadows.
v0.6 adds real shadows, reusing the v0.5 pass/render-to-texture machinery:

1. **Shadow pass** — render scene depth from the light's orthographic POV into a
   depth-only texture (a new `Framebuffer` mode).
2. **Light UBO** — the light-space matrix + direction at binding 1, uploaded once
   per frame.
3. **PCF sampling** — the mesh shader projects each fragment into light space and
   compares against the shadow map with a slope-scaled bias and 3×3 PCF.
4. **Ground plane** — a flat receiver mesh so the swarm's shadows are visible.

Shadow mapping is another render-to-texture consumer: the shadow pass writes a
depth texture; the scene and minimap passes sample it.

## 2. Roadmap context (this spec is v0.6 only)

| Milestone | Adds |
| --- | --- |
| v0.1–v0.5 (done) | Command buffer, sort keys, mesh path, JobSystem, camera UBO, HDR framebuffer + tonemap, render passes + RTT minimap. |
| **v0.6 (this spec)** | Depth-only `Framebuffer`, Light UBO (binding 1), directional shadow map + shadow pass, PCF mesh-shader shadows, ground-plane receiver. |
| Later | Multiple lights, point/spot shadows, cascaded shadow maps, game-facing pass/light API, MRT G-buffer + deferred, normal/roughness materials, Vulkan backend. |

## 3. Decisions (locked)

| Decision | Choice |
| --- | --- |
| Light | Single directional light; direction default `normalize(0.4, 0.8, 0.3)`; `Renderer::setLightDirection(vec3)` |
| Shadow projection | Orthographic from the light; scene `center (0, 0.5, 0)`, `halfExtent 16`, `near 0.1`, `far 64` (eye placed `far*0.5` behind the scene) |
| Shadow map | Depth-only `Framebuffer`, 2048×2048, `DEPTH_COMPONENT32F`, hardware compare (`sampler2DShadow`) |
| Filtering | 3×3 PCF (9 taps, `1/2048` texel) with a slope-scaled bias `max(0.0025*(1-N·L), 0.0008)` |
| Light UBO | `LightUniforms { mat4 lightViewProj; vec4 lightDir; }`, std140, binding **1** |
| Shadow map unit | Bound to texture unit 1; mesh shader `u_shadowMap = 1` |
| Receiver | A ground-plane mesh (unit cube scaled ~40×0.1×40 at `y = -0.05`); grid stays as context |
| Backend | `executeShadow(entries, lightViewProj, ...)` + `setLight(...)`; `execute` binds the shadow map |
| Minimap | Kept; its `execute` also samples the shadow map |
| DoD | Swarm casts soft (PCF) shadows onto the ground plane and each other |

## 5. Depth-only Framebuffer

`FramebufferSpec` gains `bool depthOnly = false`. `OpenGLFramebuffer` when
`depthOnly`:

- Creates a `DEPTH_COMPONENT32F` depth texture (no color texture),
  `glNamedFramebufferTexture(GL_DEPTH_ATTACHMENT, depthTex)`, and
  `glNamedFramebufferDrawBuffer(GL_NONE)` + `glNamedFramebufferReadBuffer(GL_NONE)`.
- Depth texture params: `GL_LINEAR` min/mag; `GL_CLAMP_TO_BORDER` with border
  `(1,1,1,1)` (outside-frustum depth = 1 = lit); `GL_TEXTURE_COMPARE_MODE =
  GL_COMPARE_REF_TO_TEXTURE`, `GL_TEXTURE_COMPARE_FUNC = GL_LEQUAL` (hardware
  PCF via `sampler2DShadow`).
- `uint32_t depthAttachment() const` returns the depth texture id;
  `colorAttachment()` returns 0.

The existing color+depth path (HDR scene, minimap) is unchanged.

## 6. Light & light-space matrix

`engine/renderer/LightUniforms.hpp` (GL-free, unit-testable):

```cpp
struct LightUniforms {           // std140, binding 1
  glm::mat4 lightViewProj{1.0f};
  glm::vec4 lightDir{0.0f};
};

// nearPlane/farPlane are positive distances; the eye is placed `farPlane*0.5`
// behind the scene along -lightDir so `center` sits mid-frustum.
inline glm::mat4 makeLightViewProj(const glm::vec3& lightDir,
                                   const glm::vec3& center, float halfExtent,
                                   float nearPlane, float farPlane) {
  const glm::vec3 dir = glm::normalize(lightDir);
  const glm::vec3 eye = center - dir * (farPlane * 0.5f);
  const glm::mat4 view = glm::lookAt(eye, center, glm::vec3(0.0f, 1.0f, 0.0f));
  const glm::mat4 proj =
      glm::ortho(-halfExtent, halfExtent, -halfExtent, halfExtent, nearPlane,
                 farPlane);
  return proj * view;
}
```

`sizeof(LightUniforms)` = 64 + 16 = 80 bytes (std140-valid). The `Renderer` builds
`LightUniforms{ makeLightViewProj(dir, {0,0.5,0}, 16, 0.1, 64), vec4(dir, 0) }`.

## 7. Backend additions

`OpenGLBackend` gains a shadow-depth program (vertex maps position by
`u_lightViewProj * u_transform`; empty fragment) and a Light UBO (binding 1):

- **`executeShadow(const std::vector<RenderEntry>&, const glm::mat4& lightViewProj,
  Arena&, const ResourceRegistry&)`**: `glEnable(GL_DEPTH_TEST)`, back-face cull;
  bind the depth program; set `u_lightViewProj`; for each `Mesh` command set
  `u_transform` and `glDrawElements` (resolve the mesh VAO from the registry).
  Non-mesh commands are skipped.
- **`setLight(const LightUniforms& light, uint32_t shadowMapTexture)`**: upload
  `light` to the Light UBO (binding 1); store `m_shadowMap = shadowMapTexture`.
- **`execute(...)`** (modified): before the mesh pass, `glBindTextureUnit(1,
  m_shadowMap)`; when the mesh shader is bound, `shader->setInt("u_shadowMap", 1)`.
  Everything else (camera UBO upload, mesh/line passes) is unchanged.

The Light UBO is created in the constructor (`UniformBuffer::Create(sizeof(
LightUniforms), 1)`). The mesh shader lives in the registry (built by the
`Renderer`); the backend only sets its `u_shadowMap` sampler + binds the texture.

## 8. `mesh.glsl` (shadow sampling)

```glsl
#type vertex
#version 460 core
layout(location = 0) in vec3 a_position;
layout(location = 1) in vec3 a_normal;
layout(std140, binding = 0) uniform Camera {
  mat4 u_view; mat4 u_projection; mat4 u_viewProjection; vec4 u_cameraPos;
};
layout(std140, binding = 1) uniform Light {
  mat4 u_lightViewProj; vec4 u_lightDir;
};
uniform mat4 u_transform;
out vec3 v_normal;
out vec4 v_lightClip;
void main() {
  v_normal = mat3(u_transform) * a_normal;
  vec4 world = u_transform * vec4(a_position, 1.0);
  v_lightClip = u_lightViewProj * world;
  gl_Position = u_viewProjection * world;
}

#type fragment
#version 460 core
in vec3 v_normal;
in vec4 v_lightClip;
layout(std140, binding = 1) uniform Light {
  mat4 u_lightViewProj; vec4 u_lightDir;
};
uniform vec4 u_baseColor;
uniform sampler2DShadow u_shadowMap;
out vec4 fragColor;

float shadowFactor(float ndl) {
  vec3 p = v_lightClip.xyz / v_lightClip.w;   // -> [-1,1]
  p = p * 0.5 + 0.5;                           // -> [0,1]
  if (p.z > 1.0) return 0.0;                    // beyond far plane: lit
  float bias = max(0.0025 * (1.0 - ndl), 0.0008);
  float texel = 1.0 / 2048.0;
  float sum = 0.0;
  for (int x = -1; x <= 1; ++x)
    for (int y = -1; y <= 1; ++y) {
      vec3 uvz = vec3(p.xy + vec2(x, y) * texel, p.z - bias);
      sum += texture(u_shadowMap, uvz);         // hardware compare, 0/1
    }
  return 1.0 - sum / 9.0;                        // 1 = fully shadowed
}

void main() {
  vec3 N = normalize(v_normal);
  vec3 L = normalize(u_lightDir.xyz);
  float ndl = max(dot(N, L), 0.0);
  float shadow = shadowFactor(ndl);
  float ambient = 0.25;
  fragColor = vec4(u_baseColor.rgb * (ambient + (1.0 - shadow) * ndl),
                   u_baseColor.a);
}
```

`texture(sampler2DShadow, vec3)` returns the hardware compare result (0..1); the
3×3 loop over hardware-compared taps gives smooth PCF edges.

## 9. Renderer & game

- `Renderer`: owns `Ref<Framebuffer> m_shadowFBO` (depth-only, 2048²), a shadow
  `RenderPass`, a `glm::vec3 m_lightDir` (default `normalize(0.4,0.8,0.3)`), and
  `LightUniforms m_light`. New `setLightDirection(const glm::vec3&)`. `endFrame`:
  compute `lightViewProj`; run the shadow pass (`beginPass` + `executeShadow`);
  `setLight`; then the existing minimap / scene / composite passes. Scene bounds
  are constants (`center (0,0.5,0)`, `halfExtent 16`, `near 0.1`, `far 64`).
- `BotArenaGame`: register a ground material (grey) and submit a ground box
  (`center (0,-0.05,0)`, `size (40,0.1,40)`) serially on the main lane, before the
  swarm. Grid stays. Uses the default light direction (no game change needed
  beyond the ground plane).

## 10. Testing

- **Unit (Catch2, no GL):** `makeLightViewProj` — transforming the scene `center`
  yields clip XY ≈ 0 (after perspective divide, which is identity for ortho) and a
  depth within `[-1,1]`; a point offset toward the light (`center - dir*k`)
  produces a smaller light-space `z` than a point offset away (`center + dir*k`)
  (i.e. nearer to the light = smaller depth).
- **Behavioral:** `BOTARENA_SCREENSHOT` shows the swarm casting soft PCF shadows
  onto the ground plane (and onto each other), lit from the directional light,
  with the minimap inset still present.

## 11. Out of scope (later milestones)

Multiple lights / a lights array, point and spot shadows (cube / perspective
shadow maps), cascaded shadow maps, a shadow atlas, a game-facing pass/light API,
MRT G-buffer + deferred lighting, normal/roughness/PBR materials, and a second
backend. The Light UBO and depth-only `Framebuffer` are designed to grow into
these without changing their surface.
