# Renderer v0.7 — MRT G-Buffer & Deferred Lighting — Design Spec

**Date:** 2026-07-01
**Status:** Approved for planning
**Scope:** Engine v0.7 — a multiple-render-target G-buffer geometry pass, a
fullscreen deferred lighting pass accumulating one shadowed directional light plus
many coloured point lights, then tonemap composite.
**Builds on:** v0.6 directional shadow maps (merged to `main`).

## 1. Goal & Motivation

Through v0.6 the scene is forward-rendered: the mesh shader lights each fragment
inline. That costs O(pixels × objects) per light and does not scale to many
lights. v0.7 switches to **deferred shading**, the destination the whole RHI was
built toward:

1. **Geometry pass** — render the scene into a multiple-render-target (MRT)
   G-buffer capturing per-pixel albedo, world normal, and world position (+ depth).
2. **Lighting pass** — a single fullscreen pass reads the G-buffer (+ shadow map +
   lights) and computes lighting once per pixel, so N lights cost O(pixels).
3. **Composite** — tonemap the lit HDR result to the screen (existing).

The payoff is demonstrated with **~16 animated coloured point lights** plus the
shadowed directional light — cheap in deferred, expensive in forward.

## 2. Roadmap context (this spec is v0.7 only)

| Milestone | Adds |
| --- | --- |
| v0.1–v0.6 (done) | Command buffer, sort keys, mesh path, JobSystem, camera UBO, HDR framebuffer + tonemap, render passes + RTT, directional shadow map. |
| **v0.7 (this spec)** | MRT `Framebuffer`, G-buffer geometry pass, fullscreen deferred lighting (1 shadowed directional + ~16 point lights), `PointLight` UBO. |
| Later | Point/spot shadows, light culling (tiled/clustered) + light volumes, SSAO/bloom, PBR (metallic/roughness) materials, deferred-aware grid/minimap, Vulkan backend. |

## 3. Decisions (locked)

| Decision | Choice |
| --- | --- |
| G-buffer | 3 MRT targets: `RGBA8` albedo, `RGBA16F` world normal, `RGBA16F` world position; + `DEPTH24_STENCIL8`; window-sized |
| Lighting | Fullscreen deferred pass: ambient `0.15`, 1 shadowed directional (3×3 PCF), up to 32 unshadowed point lights |
| Point lights | ~16 animated coloured lights; `PointLight { vec4 positionRadius; vec4 color; }`; UBO binding 2 `{ int count; PointLight lights[32]; }` |
| Shadow | Directional only (v0.6 shadow map); PCF moves from the mesh shader into the lighting pass |
| Scene | Meshes only — ground + walls + swarm; grid, debug line, and minimap dropped this milestone |
| MRT Framebuffer | `FramebufferSpec::colorFormats` (enum list); `colorAttachment(uint32_t index = 0)` |
| Backend interface | `execute` → `executeGeometry`; add `setPointLights`, `lightingPass`; keep `beginPass`/`executeShadow`/`setLight`/`blit`/`readPixels` |
| DoD | Swarm + ground lit through the deferred G-buffer by the shadowed directional + multiple coloured point lights, tonemapped |

## 4. Architecture & data flow

```
BotArenaGame::onRender
  │  renderer.setCamera(*mainCamera)
  │  renderer.setPointLights(animated ring of ~16 coloured lights)
  │  serial: ground + walls; parallel: swarm            (all Mesh commands)
  ▼
Renderer::endFrame (merge + stable_sort once)
  │  lightViewProj = makeLightViewProj(...)
  │  // shadow pass
  │  beginPass(shadowFBO, {}, clearDepth, 2048, 2048); executeShadow(merged, lightViewProj)
  │  setLight({lightViewProj, dir}, shadowFBO->depthAttachment())
  │  // geometry pass -> G-buffer MRT
  │  beginPass(gbufferFBO, clear, clearDepth, w, h); executeGeometry(merged, camera)
  │  // lighting pass -> HDR scene FBO (fullscreen; reads G-buffer + shadow + lights)
  │  beginPass(sceneFBO, {0,0,0,1}, noDepth, w, h)
  │  lightingPass(gAlbedo, gNormal, gWorldPos, shadowMap)
  │  // composite -> default framebuffer (tonemap)
  │  beginPass(nullptr, {0,0,0,1}, noDepth, w, h); blit(sceneFBO->colorAttachment(), fullscreen)
  ▼
saveScreenshot -> readPixels(default framebuffer)
```

## 5. MRT Framebuffer

`engine/renderer/Framebuffer.hpp`:

```cpp
enum class FramebufferFormat { RGBA8, RGBA16F };

struct FramebufferSpec {
  uint32_t width = 1280;
  uint32_t height = 720;
  bool hdr = true;                                 // used when colorFormats is empty
  bool depthOnly = false;
  std::vector<FramebufferFormat> colorFormats;     // empty => single (hdr ? RGBA16F : RGBA8)
};

class Framebuffer {
  ...
  virtual uint32_t colorAttachment(uint32_t index = 0) const = 0;
  virtual uint32_t depthAttachment() const = 0;
};
```

`OpenGLFramebuffer` (`invalidate`): if `depthOnly`, the v0.6 path. Otherwise build
the color attachments — if `colorFormats` is non-empty, one texture per format;
else a single texture (`hdr ? RGBA16F : RGBA8`) as today. Attach
`GL_COLOR_ATTACHMENT0 + i`; if more than one, call `glNamedFramebufferDrawBuffers(n,
{GL_COLOR_ATTACHMENT0..})`. Store the texture ids in a `std::vector<uint32_t>
m_colors`; `colorAttachment(i)` returns `m_colors[i]`. A `DEPTH24_STENCIL8`
depth-stencil texture as today. `resize` recreates all attachments.

The G-buffer spec: `colorFormats = { RGBA8, RGBA16F, RGBA16F }` (albedo, normal,
world position). The scene/composite framebuffers keep the single-color path.

## 6. Geometry pass — `mesh.glsl` writes the G-buffer

`mesh.glsl` becomes the geometry shader (no lighting, no shadow sampling):

```glsl
#type vertex
#version 460 core
layout(location = 0) in vec3 a_position;
layout(location = 1) in vec3 a_normal;
layout(std140, binding = 0) uniform Camera {
  mat4 u_view; mat4 u_projection; mat4 u_viewProjection; vec4 u_cameraPos;
};
uniform mat4 u_transform;
out vec3 v_worldNormal;
out vec3 v_worldPos;
void main() {
  vec4 world = u_transform * vec4(a_position, 1.0);
  v_worldPos = world.xyz;
  v_worldNormal = mat3(u_transform) * a_normal;
  gl_Position = u_viewProjection * world;
}

#type fragment
#version 460 core
in vec3 v_worldNormal;
in vec3 v_worldPos;
uniform vec4 u_baseColor;
layout(location = 0) out vec4 gAlbedo;
layout(location = 1) out vec4 gNormal;
layout(location = 2) out vec4 gWorldPos;
void main() {
  gAlbedo = u_baseColor;
  gNormal = vec4(normalize(v_worldNormal), 1.0);
  gWorldPos = vec4(v_worldPos, 1.0);
}
```

`backend.executeGeometry(entries, camera, arena, registry)` is the v0.6 mesh pass
(upload camera UBO; back-face cull; bind material shaders; `glDrawElements` per
`Mesh` command) writing into the bound MRT G-buffer. No line pass, no shadow-map
binding.

## 7. Lighting pass

`engine/renderer/PointLight.hpp`:

```cpp
struct PointLight {
  glm::vec4 positionRadius{0.0f};   // xyz position, w radius
  glm::vec4 color{1.0f};            // rgb colour, a intensity
};
```

`backend.setPointLights(int count, const PointLight* lights)` uploads a UBO at
binding 2 laid out as `{ int count; PointLight lights[32]; }` (std140; the C++ side
mirrors it with padding: `struct { int32_t count; int32_t pad[3]; PointLight
lights[32]; }`, 1040 bytes). Count is clamped to 32.

`backend.lightingPass(uint32_t gAlbedo, uint32_t gNormal, uint32_t gWorldPos,
uint32_t shadowMap)` binds those four textures to units 0–3, binds the lighting
program + the fullscreen quad, and draws into the currently bound target (the HDR
scene FBO). The lighting shader (backend-owned, inline):

- reads `Light` UBO (binding 1) and `PointLights` UBO (binding 2);
- samples `gAlbedo/gNormal/gWorldPos` at the fragment;
- ambient `0.15 * albedo`;
- directional: `max(N·L,0) * albedo * (1 - shadowPCF(worldPos))`, where
  `shadowPCF` projects `worldPos` by `u_lightViewProj` and does the 3×3 PCF against
  `u_shadowMap` (the code moved out of the mesh shader);
- for each of `u_pointCount` lights: `att * max(N·Lp,0) * color.rgb * color.a *
  albedo`, with `att = clamp(1 - d/radius, 0, 1)^2` (`d` = distance to the light);
- writes the summed HDR colour.

The lighting pass runs with depth test off (fullscreen); the composite `blit`
(Reinhard + gamma) is unchanged.

## 8. Renderer & game

- `Renderer` owns `m_gbufferFBO` (MRT: `{RGBA8, RGBA16F, RGBA16F}` + depth,
  resized with the window) alongside `m_sceneFBO` (HDR lit result) and
  `m_shadowFBO`. `endFrame` runs shadow → geometry → lighting → composite (§4).
  `setPointLights(const std::vector<PointLight>&)` forwards to the backend
  (clamped to 32). The minimap framebuffer/pass/camera members are removed.
- `BotArenaGame`: each frame builds ~16 `PointLight`s on a slowly rotating ring
  (`pointLightRing(index, count, time, radius, height)` for positions, a small
  colour palette, fixed radius/intensity) and calls `renderer.setPointLights(...)`.
  Submits ground + walls (serial) + swarm (parallel) as `Mesh` commands only — the
  `DebugRenderer` grid + bot-line and the `setMinimapCamera` call are removed. The
  directional light keeps its default direction.

## 9. Testing

- **Unit (Catch2, no GL):** `pointLightRing(index, count, time, radius, height)` —
  returns positions evenly spaced on a circle: all at `y == height` and distance
  `radius` from the origin in XZ; the angle between consecutive indices is
  `2π/count`; `time` rotates the whole ring rigidly (a shared phase offset).
- **Behavioral:** `BOTARENA_SCREENSHOT` shows the swarm + ground rendered through
  the deferred G-buffer and lit by the shadowed directional light plus multiple
  coloured point lights (visible coloured light pools on the ground and cubes),
  tonemapped.

## 10. Out of scope (later milestones)

Point/spot-light shadows, light culling (tiled/clustered) and light-volume
rendering (all point lights currently shade every pixel — fine at ~16), SSAO,
bloom and other G-buffer post-processing, PBR metallic/roughness materials,
returning the grid/minimap as deferred-aware views, and a second backend. The MRT
`Framebuffer`, `PointLight` UBO, and geometry/lighting split are designed to grow
into these without changing their surface.
