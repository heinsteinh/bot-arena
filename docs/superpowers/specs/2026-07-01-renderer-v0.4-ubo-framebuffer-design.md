# Renderer v0.4 — Uniform Buffer & HDR Framebuffer — Design Spec

**Date:** 2026-07-01
**Status:** Approved for planning
**Scope:** Engine v0.4 — a shared camera `UniformBuffer`, an offscreen HDR
`Framebuffer`, and a fullscreen tonemap composite pass. The scene renders into an
HDR target and is composited to the screen.
**Builds on:** v0.3 job system + swarm (merged to `main`).

## 1. Goal & Motivation

Through v0.3 the backend renders straight to the default framebuffer and sets
`u_viewProjection` as a plain per-shader uniform. v0.4 adds the two RHI resources
that unlock post-processing and (later) deferred shading:

1. **`UniformBuffer`** — a single camera block (`CameraUniforms`) uploaded once
   per frame and read by every shader at binding 0, replacing per-shader
   `u_viewProjection`.
2. **`Framebuffer`** — an offscreen HDR (`RGBA16F`) color + depth target. The
   scene renders into it; a fullscreen **composite** pass then tonemaps (Reinhard
   + gamma) it to the default framebuffer.

The composite is the screen-space seam future post-processing and the deferred
lighting pass build on. Both resources are proven end-to-end by the existing
swarm scene rendered through the new HDR pipeline.

## 2. Roadmap context (this spec is v0.4 only)

| Milestone | Adds |
| --- | --- |
| v0.1–v0.3 (done) | Command buffer, sort keys, mesh path, ResourceRegistry, JobSystem + thread-local arenas. |
| **v0.4 (this spec)** | Camera `UniformBuffer` (binding 0), offscreen HDR `Framebuffer`, fullscreen tonemap composite. |
| v0.5 | Extract `RenderPass{target, clear}`; expose render-to-texture; MRT G-buffer. |
| Later | Deferred lighting, bloom/exposure, GPU instancing, Vulkan backend. |

## 3. Decisions (locked)

| Decision | Choice |
| --- | --- |
| Resources | `UniformBuffer` + `Framebuffer`, abstract base + `OpenGL*` impl + `Ref<>`/`Create()` (matches `Buffer`/`Shader`) |
| Orchestration | Offscreen bind + composite live **inside `OpenGLBackend::beginFrame/endFrame`**; `Renderer` + game APIs stay game-facing |
| Camera block | `CameraUniforms { mat4 view, projection, viewProjection; vec4 cameraPosition; }` (std140, 208 B) at binding 0 |
| Camera position | Recovered from the view matrix (`inverse(view)[3]`) — no new `Camera` API; populated but unused by shaders in v0.4 |
| HDR format | Offscreen color `RGBA16F`, depth `DEPTH24_STENCIL8` |
| Composite | Fullscreen triangle, Reinhard tonemap + gamma 2.2 |
| Renderer API | `setViewProjection(mat4)` → `setCamera(const Camera&)`; everything else unchanged |
| Resize | `beginFrame` resizes the offscreen framebuffer to the window size |
| RenderPass | Deferred to v0.5 (not introduced here) |
| DoD | Swarm rendered through HDR offscreen + tonemap composite; `makeCameraUniforms` unit-tested |

## 4. Architecture & data flow

```
BotArenaGame::onRender
  │  renderer.setCamera(*camera)   // builds CameraUniforms
  │  serial grid + walls -> renderer.queue()
  │  renderer.generateMeshes(kBotCount, batchFn)   // swarm (unchanged)
  ▼
Renderer::beginFrame(w,h)  -> backend.beginFrame(w,h)
                                m_sceneFBO->resize(w,h); m_sceneFBO->bind();
                                depth on; clear
  (layers submit into per-lane queues)
Renderer::endFrame
  │  merge lanes + stable_sort
  │  backend.execute(merged, cameraUniforms, arena, registry)
  │     -> m_cameraUBO->setData(cameraUniforms)   // once per frame, binding 0
  │     -> draw scene into the bound HDR FBO (shaders read the UBO)
  │  backend.endFrame()
  │     -> m_sceneFBO->unbind(); viewport; clear; depth off
  │     -> bind m_sceneFBO->colorAttachment(); draw fullscreen tri
  │        with composite shader (Reinhard + gamma) -> default framebuffer
  ▼
saveScreenshot -> readPixels(default framebuffer)   // the composited image
```

Only the `execute` parameter type changes on the `RenderBackend` interface
(`const glm::mat4&` → `const CameraUniforms&`); `beginFrame`/`endFrame`/
`readPixels` keep their shapes. The `Renderer` and game are otherwise unchanged
apart from `setCamera`.

## 5. UniformBuffer & CameraUniforms

`engine/renderer/UniformBuffer.hpp` (abstract) + `opengl/OpenGLUniformBuffer`:

```cpp
class UniformBuffer {
 public:
  virtual ~UniformBuffer() = default;
  virtual void setData(const void* data, uint32_t size, uint32_t offset = 0) = 0;
  static Ref<UniformBuffer> Create(uint32_t size, uint32_t binding);
};
```

`OpenGLUniformBuffer(size, binding)`: `glCreateBuffers` +
`glNamedBufferData(id, size, nullptr, GL_DYNAMIC_DRAW)` +
`glBindBufferBase(GL_UNIFORM_BUFFER, binding, id)`. `setData` →
`glNamedBufferSubData`.

`engine/renderer/CameraUniforms.hpp` (GL-free, unit-testable):

```cpp
struct CameraUniforms {
  glm::mat4 view{1.0f};
  glm::mat4 projection{1.0f};
  glm::mat4 viewProjection{1.0f};
  glm::vec4 cameraPosition{0.0f};
};

inline CameraUniforms makeCameraUniforms(const glm::mat4& view,
                                         const glm::mat4& projection) {
  CameraUniforms u;
  u.view = view;
  u.projection = projection;
  u.viewProjection = projection * view;
  u.cameraPosition = glm::vec4(glm::vec3(glm::inverse(view)[3]), 1.0f);
  return u;
}
```

`sizeof(CameraUniforms)` = 3·64 + 16 = 208 bytes, a valid std140 layout (all
members 16-byte aligned, no interior padding). `cameraPosition` is populated but
no v0.4 shader reads it (reserved for future specular/PBR).

## 6. Framebuffer

`engine/renderer/Framebuffer.hpp` (abstract) + `opengl/OpenGLFramebuffer`:

```cpp
struct FramebufferSpec {
  uint32_t width = 1280;
  uint32_t height = 720;
  bool hdr = true;
};

class Framebuffer {
 public:
  virtual ~Framebuffer() = default;
  virtual void bind() = 0;      // bind FBO + set viewport to its size
  virtual void unbind() = 0;    // bind default framebuffer (0)
  virtual void resize(uint32_t width, uint32_t height) = 0;
  virtual uint32_t colorAttachment() const = 0;  // GL texture id
  static Ref<Framebuffer> Create(const FramebufferSpec& spec);
};
```

`OpenGLFramebuffer` (DSA): color texture `RGBA16F` (hdr) or `RGBA8`, linear
filtering, clamp-to-edge; depth-stencil texture `DEPTH24_STENCIL8`;
`glNamedFramebufferTexture` for color attachment 0 + depth-stencil;
`glCheckNamedFramebufferStatus` completeness check (log on failure). `resize`
skips when the size is unchanged or zero, else destroys and recreates the
attachments. `bind` sets `glViewport(0,0,w,h)`.

## 7. Backend changes

`OpenGLBackend` ctor additionally creates: `m_cameraUBO =
UniformBuffer::Create(sizeof(CameraUniforms), 0)`, `m_sceneFBO =
Framebuffer::Create({...hdr=true})`, a composite shader program (inline GLSL), and
a fullscreen-triangle VAO/VBO.

- **`beginFrame(int w, int h)`**: `m_sceneFBO->resize(w,h)`; `m_sceneFBO->bind()`;
  `glEnable(GL_DEPTH_TEST)`; `glClearColor(0.08,0.09,0.11,1)`; `glClear(color|depth)`.
- **`execute(const std::vector<RenderEntry>&, const CameraUniforms& camera,
  Arena&, const ResourceRegistry&)`**: `m_cameraUBO->setData(&camera,
  sizeof(CameraUniforms))` once; then the v0.3 mesh pass + line pass unchanged,
  except neither sets `u_viewProjection` (shaders read the UBO).
- **`endFrame()`**: `m_sceneFBO->unbind()`; `glViewport(0,0,m_width,m_height)`;
  `glDisable(GL_DEPTH_TEST)`; `glClear(GL_COLOR_BUFFER_BIT)`;
  `glBindTextureUnit(0, m_sceneFBO->colorAttachment())`; bind composite program,
  set `u_scene=0`; draw the fullscreen triangle (`glDrawArrays(GL_TRIANGLES,0,3)`);
  re-enable depth test for the next frame. The backend tracks `m_width/m_height`
  from `beginFrame`.
- **`readPixels`** unchanged — the default framebuffer now holds the composited
  image.

The fullscreen triangle uses `gl_VertexID`-free explicit geometry (3 verts,
position + uv) so it flows through the same simple attrib setup; the composite
shader samples `u_scene`.

## 8. Shaders

- `assets/shaders/mesh.glsl` — replace `uniform mat4 u_viewProjection;` with:
  ```glsl
  layout(std140, binding = 0) uniform Camera {
    mat4 u_view;
    mat4 u_projection;
    mat4 u_viewProjection;
    vec4 u_cameraPos;
  };
  ```
  The vertex main still uses `u_viewProjection * u_transform * vec4(a_position,1)`.
- **Inline line shader** (backend GLSL) — add the same `Camera` UBO block, drop
  the `u_viewProjection` uniform, use `u_viewProjection` from the block.
- **Inline composite shader** (backend GLSL) — vertex passes through
  position + uv; fragment:
  ```glsl
  vec3 c = texture(u_scene, v_uv).rgb;
  c = c / (c + vec3(1.0));          // Reinhard
  c = pow(c, vec3(1.0 / 2.2));      // gamma
  fragColor = vec4(c, 1.0);
  ```

## 9. Renderer & game changes

- `Renderer`: replace `setViewProjection(const glm::mat4&)` with
  `setCamera(const Camera& camera)` storing
  `m_camera = makeCameraUniforms(camera.view(), camera.projection())`; `endFrame`
  passes `m_camera` to `backend->execute`. `generateMeshes`, per-lane buffers,
  `queue()`, `registry()`, `unitCubeMesh()`, `meshShader()`, `saveScreenshot`
  unchanged. Include `CameraUniforms.hpp` + `Camera.hpp`.
- `BotArenaGame::onRender`: `renderer.setViewProjection(camera->viewProjection())`
  → `renderer.setCamera(*camera)`. Nothing else changes.

## 10. Testing

- **Unit (Catch2, no GL):** `makeCameraUniforms` — for an identity view,
  `viewProjection == projection` and `cameraPosition == (0,0,0,1)`; for a
  `glm::lookAt(eye, center, up)` view, `viewProjection == projection * view` and
  `cameraPosition.xyz ≈ eye` (recovered from `inverse(view)`).
- **Behavioral:** `BOTARENA_SCREENSHOT` renders the swarm through the HDR
  offscreen target and the tonemap composite. The image differs from v0.3 — it is
  Reinhard-tonemapped and gamma-corrected (brighter mid-tones, rolled-off
  highlights) — which is the expected result of the HDR pipeline; it is still
  clearly the swarm in the arena. The fixed 1280×720 screenshot exercises the
  framebuffer resize path (first-frame allocation to window size).

## 11. Out of scope (later milestones)

`RenderPass{target, clear}` extraction, exposing the framebuffer for game
render-to-texture, MRT G-buffer / deferred lighting, bloom / exposure / other
post effects, GPU instancing, using `cameraPosition` for specular/PBR, and a
second backend. `UniformBuffer` and `Framebuffer` are designed to accept these
without changing their public surface.
