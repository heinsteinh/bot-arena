# Renderer v0.9 — Environment Cubemap & Skybox — Design Spec

**Date:** 2026-07-01
**Status:** Approved for planning
**Scope:** Engine v0.9 — a `TextureCube` RHI resource, a procedural sky rendered
into an environment cubemap, and that cubemap drawn as the scene background
(skybox). The reusable foundation for image-based lighting.
**Builds on:** v0.8 PBR materials (merged to `main`).

## 1. Goal & Motivation

v0.8's PBR uses a flat constant ambient because there is no environment to reflect.
Image-based lighting fixes that, but IBL is large, so it is staged:

- **v0.9 (this spec):** build the environment infrastructure — a cubemap resource,
  a procedural sky rendered into it, and shown as the background — a visible,
  shippable increment.
- **v0.10 (later):** the IBL itself — irradiance convolution, prefiltered
  environment map, BRDF LUT, and swap the constant ambient for IBL.

v0.9 delivers the `TextureCube` type and the render-into-each-cubemap-face
primitive that v0.10's convolution passes reuse, plus a visible sky.

## 2. Roadmap context (this spec is v0.9 only)

| Milestone | Adds |
| --- | --- |
| v0.1–v0.8 (done) | Command buffer, sort/mesh, jobs, camera UBO, HDR + tonemap, passes/RTT, shadow map, deferred G-buffer, PBR (Cook-Torrance). |
| **v0.9 (this spec)** | `TextureCube` resource, render-to-cubemap-face, procedural sky env map, skybox background, `invViewProjection` in the camera UBO. |
| v0.10 | Irradiance convolution, prefiltered env + BRDF LUT, ambient → IBL. |
| Later | `.hdr` environment loading, rotatable/animated sky, bloom/SSAO, light culling. |

## 3. Decisions (locked)

| Decision | Choice |
| --- | --- |
| Environment | Procedural gradient sky (zenith → horizon → ground) + a sun disc along the directional light; no external asset |
| Cubemap resource | `TextureCube` (GL_TEXTURE_CUBE_MAP, RGBA16F, linear, clamp-to-edge, seamless); env map 512², 1 mip |
| Env generation | Backend renders a fullscreen sky pass into each of the 6 faces at Renderer init |
| Skybox | Lighting pass: background pixels (`dot(N,N) < 0.5`) sample the env cubemap by the reconstructed world view ray |
| View ray | `CameraUniforms` gains `invViewProjection`; the lighting shader unprojects the pixel NDC |
| Env binding | Env cubemap bound to lighting-pass texture unit 4 (`samplerCube u_envMap`) via `setEnvironment` |
| Reflections | Out of scope (v0.10) — v0.9 shows the sky only, metals do not reflect it |
| DoD | A procedural sky (gradient + sun) fills the background behind the PBR-lit scene |

## 4. Architecture & data flow

```
Renderer ctor
  │  m_envMap = TextureCube::Create(512, 1)
  │  backend.renderEnvironment(m_envMap->rendererID(), 512, m_lightDir)   // 6 faces, sky shader
  │  backend.setEnvironment(m_envMap->rendererID())
  ▼
Renderer::endFrame (per frame, unchanged structure)
  shadow -> geometry -> lighting -> composite
                          │
                          ▼
  Lighting pass (fullscreen)   [Camera UBO binding 0 now carries invViewProjection]
    if (dot(N,N) < 0.5) {                         // background pixel
      clip = vec4(v_uv*2-1, 1, 1)
      world = u_invViewProjection * clip
      dir = normalize(world.xyz/world.w - u_cameraPos.xyz)
      fragColor = texture(u_envMap, dir)          // skybox
      return
    }
    ... PBR shading as v0.8 (unchanged) ...
```

## 5. `TextureCube` resource

`engine/renderer/TextureCube.hpp` (abstract) + `opengl/OpenGLTextureCube.hpp/.cpp`:

```cpp
class TextureCube {
 public:
  virtual ~TextureCube() = default;
  virtual void bind(uint32_t unit) const = 0;
  virtual uint32_t rendererID() const = 0;
  static Ref<TextureCube> Create(uint32_t size, uint32_t mipLevels = 1);
};
```

`OpenGLTextureCube`: `glCreateTextures(GL_TEXTURE_CUBE_MAP, 1, &id)`,
`glTextureStorage2D(id, mipLevels, GL_RGBA16F, size, size)` (allocates all 6
faces), min/mag `GL_LINEAR`, wrap `GL_CLAMP_TO_EDGE` on S/T/R,
`GL_TEXTURE_CUBE_MAP_SEAMLESS` is a global enable set once in the backend ctor.
`bind(unit)` = `glBindTextureUnit(unit, id)`. The destructor deletes the texture.

## 6. Camera UBO — `invViewProjection`

`CameraUniforms` gains a trailing `mat4 invViewProjection` (std140 offset 208,
after `cameraPosition`; earlier offsets unchanged, so `mesh.glsl`'s Camera block —
which does not declare it — is unaffected):

```cpp
struct CameraUniforms {
  glm::mat4 view{1.0f};
  glm::mat4 projection{1.0f};
  glm::mat4 viewProjection{1.0f};
  glm::vec4 cameraPosition{0.0f};
  glm::mat4 invViewProjection{1.0f};
};
// makeCameraUniforms: u.invViewProjection = glm::inverse(u.viewProjection);
```

The backend's camera UBO already sizes to `sizeof(CameraUniforms)`, so it grows
automatically.

## 7. Render-to-cubemap + procedural sky (backend)

Two new `RenderBackend` methods:

```cpp
virtual void renderEnvironment(uint32_t cubemap, int size,
                               const glm::vec3& sunDir) = 0;
virtual void setEnvironment(uint32_t envCubemap) = 0;
```

- **`renderEnvironment`** creates a temporary FBO, and for each of the 6 faces:
  `glNamedFramebufferTextureLayer(fbo, GL_COLOR_ATTACHMENT0, cubemap, 0, face)`,
  `glViewport(0,0,size,size)`, binds the sky program with that face's
  `forward`/`right`/`up` basis and `sunDir`, and draws the unit quad (`m_quadVao`).
  The FBO is deleted afterward. The 6 face bases follow the GL cubemap convention
  (`+X,-X,+Y,-Y,+Z,-Z` with the standard `-Y` up-vectors).
- **`setEnvironment`** stores the cubemap id in `m_envMap` for the lighting pass to
  bind.

The **sky program** (backend-owned, inline): the vertex stage is the fullscreen
quad; the fragment reconstructs its world direction
`dir = normalize(forward + (uv.x*2-1)*right + (uv.y*2-1)*up)` and outputs an HDR
sky:

```glsl
float h = dir.y;
vec3 zenith  = vec3(0.25, 0.45, 0.85);
vec3 horizon = vec3(0.75, 0.82, 0.95);
vec3 ground  = vec3(0.15, 0.13, 0.11);
vec3 sky = (h > 0.0) ? mix(horizon, zenith, pow(clamp(h, 0.0, 1.0), 0.5))
                     : mix(horizon, ground, clamp(-h * 3.0, 0.0, 1.0));
float s = max(dot(dir, normalize(sunDir)), 0.0);
sky += vec3(1.0, 0.95, 0.85) * pow(s, 512.0) * 30.0;   // sun disc (HDR)
sky += vec3(1.0, 0.9, 0.7) * pow(s, 8.0) * 0.3;        // soft glow
fragColor = vec4(sky, 1.0);
```

## 8. Skybox in the lighting pass

`OpenGLBackend::lightingPass` binds the stored env cubemap to unit 4. The lighting
fragment shader adds `layout(std140, binding = 0) uniform Camera { ...;
mat4 u_invViewProjection; };`, a `uniform samplerCube u_envMap;`, and replaces the
flat background branch with the skybox sample (see §4). The PBR path for geometry
pixels is unchanged from v0.8.

## 9. Renderer & game

- `Renderer` owns `Ref<TextureCube> m_envMap`. In the constructor (after the
  backend and framebuffers exist) it creates the cubemap, calls
  `renderEnvironment(m_envMap->rendererID(), kEnvSize, m_lightDir)`, and
  `setEnvironment(m_envMap->rendererID())`. `kEnvSize = 512`. The env is static
  (generated once); `endFrame` is otherwise unchanged.
- `BotArenaGame`: no change — the scene renders against the new sky, and the
  skybox sun matches the default directional light.

## 10. Testing

- **Unit (Catch2, no GL):** `makeCameraUniforms` fills `invViewProjection` — assert
  `invViewProjection * viewProjection ≈ identity` (per-element, margin 1e-3) for a
  representative view+perspective pair.
- **Behavioral:** `BOTARENA_SCREENSHOT` shows a procedural sky (blue-to-pale
  gradient with a bright sun in the light direction) filling the background behind
  the swarm, the scene still PBR-lit as in v0.8. The metals do **not** yet reflect
  the sky.

## 11. Out of scope (v0.10 and later)

Irradiance convolution (diffuse ambient), prefiltered environment map + BRDF LUT
(specular ambient), and swapping the constant ambient for IBL — all v0.10, and all
reusing v0.9's `TextureCube` + `renderEnvironment` primitive. Later: `.hdr`
environment loading, a rotatable/animated sky, bloom/SSAO, light culling.
