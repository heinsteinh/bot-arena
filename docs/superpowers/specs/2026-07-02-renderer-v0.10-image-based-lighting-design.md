# Renderer v0.10 — Image-Based Lighting (Split-Sum) — Design Spec

**Date:** 2026-07-02
**Status:** Approved for planning
**Scope:** Engine v0.10 — precompute an irradiance cubemap, a prefiltered
environment cubemap, and a BRDF integration LUT from the v0.9 environment cubemap,
and replace the constant PBR ambient with split-sum image-based lighting.
**Builds on:** v0.9 environment cubemap + skybox (merged to `main`).

## 1. Goal & Motivation

v0.8's PBR uses a constant ambient because there was nothing to reflect; v0.9 added
an environment cubemap + skybox but no reflections. v0.10 finishes IBL with the
standard **split-sum approximation** (Karis/Epic): three maps precomputed once from
the environment, then sampled per-pixel to give physically based ambient — diffuse
sky fill and specular environment reflections. Metals finally mirror the sky.

## 2. Roadmap context (this spec is v0.10 only)

| Milestone | Adds |
| --- | --- |
| v0.1–v0.9 (done) | …, deferred G-buffer, PBR (Cook-Torrance), environment cubemap + skybox. |
| **v0.10 (this spec)** | Irradiance cubemap, prefiltered env cubemap, BRDF LUT, ambient → split-sum IBL. |
| Later | `.hdr` environment loading, dynamic env re-convolution, AO channel / SSAO into ambient, multi-scatter compensation, bloom, light culling. |

## 3. Decisions (locked)

| Decision | Choice |
| --- | --- |
| Method | Split-sum IBL: irradiance (diffuse) + prefiltered env & BRDF LUT (specular) |
| Irradiance map | `TextureCube` 32², 1 mip; cosine-hemisphere convolution of the env (delta 0.025) |
| Prefilter map | `TextureCube` 128² base, 5 mips; GGX importance sampling (1024 samples), roughness = mip/(mips−1) |
| BRDF LUT | 512² `Framebuffer` (RGBA16F, sample `.rg`) — reuse `Framebuffer`, no new resource; fullscreen split-sum integration |
| Texture units | env 4 (v0.9), irradiance 5, prefilter 6, BRDF LUT 7 |
| Ambient | `kD·(irradiance·albedo) + prefiltered·(F·brdf.x + brdf.y)`; `F = fresnelSchlickRoughness(NoV,F0,rough)`; AO = 1 |
| Unchanged | Direct-light Cook-Torrance and the skybox stay exactly as v0.9 |
| Generation | Once at Renderer init, after the env cubemap; `setIBL(irradiance, prefilter, brdfLUT, mips)` |
| Face rendering | Reuse v0.9's per-face fullscreen + basis approach; add render-to-cubemap-**mip**-face for the prefilter |
| Testing | No new unit test (shader-only math; CPU duplication would break DRY); existing 34 tests stay green; behavioural screenshot |
| DoD | Metals reflect the sky (gradient + sun); sky-coloured ambient; rough surfaces get blurred reflections |

## 4. Architecture & data flow

```
Renderer ctor (once, after env cubemap from v0.9)
  │  m_irradianceMap = TextureCube::Create(32, 1)
  │  m_prefilterMap  = TextureCube::Create(128, 5)
  │  m_brdfFBO       = Framebuffer::Create({512, 512})            // RGBA16F
  │  backend.convolveIrradiance(env, m_irradianceMap->rendererID(), 32)
  │  backend.prefilterEnvironment(env, m_prefilterMap->rendererID(), 128, 5)
  │  backend.beginPass(m_brdfFBO, {0,0,0,1}, false, 512, 512)
  │  backend.integrateBRDF()
  │  backend.setIBL(m_irradianceMap->rendererID(), m_prefilterMap->rendererID(),
  │                 m_brdfFBO->colorAttachment(), 5)
  ▼
Lighting pass (fullscreen)   [binds env 4, irradiance 5, prefilter 6, brdfLUT 7]
  ... skybox for background pixels (v0.9, unchanged) ...
  ... direct Cook-Torrance for dir + point lights (v0.8, unchanged) ...
  ambient (replaces the v0.8 constant term):
    F  = fresnelSchlickRoughness(NoV, F0, rough)
    kD = (1 - F) * (1 - metallic)
    diffuse  = texture(u_irradiance, N).rgb * albedo
    R        = reflect(-V, N)
    prefilt  = textureLod(u_prefilter, R, rough * float(u_prefilterMips - 1)).rgb
    ab       = texture(u_brdfLUT, vec2(NoV, rough)).rg
    specular = prefilt * (F * ab.x + ab.y)
    color    = kD * diffuse + specular
```

## 5. Backend — convolution shaders + methods

New `RenderBackend` methods:

```cpp
virtual void convolveIrradiance(uint32_t env, uint32_t irradianceCube,
                                int size) = 0;
virtual void prefilterEnvironment(uint32_t env, uint32_t prefilterCube,
                                  int baseSize, int mipCount) = 0;
virtual void integrateBRDF() = 0;                         // into the bound FBO
virtual void setIBL(uint32_t irradiance, uint32_t prefilter, uint32_t brdfLUT,
                    int prefilterMips) = 0;
```

`OpenGLBackend` owns three inline programs (`m_irradianceShader`,
`m_prefilterShader`, `m_brdfShader`) and stores `m_irradianceMap`,
`m_prefilterMap`, `m_brdfLUT`, `m_prefilterMips`.

- The 6-face basis table used by `renderEnvironment` is extracted to an
  anonymous-namespace constant `kCubeFaces` and reused by the two convolution
  methods (DRY).
- **`convolveIrradiance`** renders the irradiance shader into the 6 faces of the
  irradiance cubemap (like `renderEnvironment`), sampling the env cubemap.
- **`prefilterEnvironment`** loops `mip = 0..mipCount-1`: viewport `baseSize >> mip`,
  `roughness = mip / (mipCount-1)`, and for each face attaches
  `glNamedFramebufferTextureLayer(fbo, COLOR0, prefilterCube, mip, face)` and draws.
- **`integrateBRDF`** draws the fullscreen BRDF shader into the currently bound FBO.
- **`setIBL`** stores the three ids + mip count; `lightingPass` binds them to units
  5/6/7 and sets `u_irradiance`/`u_prefilter`/`u_brdfLUT`/`u_prefilterMips`.

### Shader math (standard split-sum)

- **Irradiance:** for the face direction `N`, build a tangent frame, integrate
  `texture(env, sampleVec).rgb * cos(theta) * sin(theta)` over the hemisphere
  (phi 0..2π, theta 0..π/2, step 0.025); result `PI * sum / sampleCount`.
- **Prefilter:** `Hammersley` + `importanceSampleGGX(Xi, N, roughness)`; with `V = N`,
  `L = reflect(-V, H)`; accumulate `texture(env, L).rgb * NdotL` weighted by `NdotL`
  over 1024 samples.
- **BRDF LUT:** fullscreen; `v_uv.x = NoV`, `v_uv.y = roughness`; the standard
  `integrateBRDF` (GGX importance sampling, geometry Smith with IBL `k =
  roughness²/2`) returning `(scale, bias)` in `.rg`.
- **`fresnelSchlickRoughness`** (lighting shader): `F0 + (max(vec3(1-rough), F0) -
  F0) * pow(1 - cosT, 5)`.

## 6. Lighting pass — ambient → IBL

The lighting fragment shader adds `samplerCube u_irradiance`, `samplerCube
u_prefilter`, `sampler2D u_brdfLUT`, `uniform int u_prefilterMips`, and
`fresnelSchlickRoughness`, then replaces the v0.8 constant-ambient block (`vec3
color = (kdAmb*albedo + Famb)*0.12;`) with the split-sum ambient (§4). The direct
Cook-Torrance accumulation and the skybox branch are untouched. `lightingPass` binds
units 5/6/7 and sets the samplers + `u_prefilterMips`.

## 7. Renderer

`Renderer` gains `Ref<TextureCube> m_irradianceMap`, `Ref<TextureCube>
m_prefilterMap`, `Ref<Framebuffer> m_brdfFBO`, and constants
`kIrradianceSize = 32`, `kPrefilterSize = 128`, `kPrefilterMips = 5`,
`kBrdfSize = 512`. In the constructor, after the env cubemap is generated, it
creates the three resources, runs the convolutions + BRDF integration, and calls
`setIBL`. `endFrame` is otherwise unchanged (the maps are static). `BotArenaGame`
does not change.

## 8. Testing

- **Unit:** none new — the convolution/importance-sampling math is entirely in
  shaders and duplicating it CPU-side would violate DRY (consistent with the
  v0.4/v0.5/v0.7/v0.9 GPU-only pieces). The existing 34 Catch2 cases must stay
  green.
- **Behavioral:** `BOTARENA_SCREENSHOT` shows the metal cubes reflecting the sky
  (blue gradient + sun), sky-coloured ambient fill replacing the flat grey, and
  rougher surfaces showing blurred environment reflections — a clear jump from v0.9.

## 9. Out of scope (later milestones)

`.hdr` environment loading, dynamic/animated environment re-convolution, an AO
channel or SSAO feeding the ambient occlusion term, multi-scatter energy
compensation, bloom, and light culling. This milestone is the standard static
split-sum IBL; those extend it.
