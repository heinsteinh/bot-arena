# Renderer v0.8 — PBR Materials (Analytic Cook-Torrance) — Design Spec

**Date:** 2026-07-01
**Status:** Approved for planning
**Scope:** Engine v0.8 — metallic/roughness materials and a Cook-Torrance BRDF in
the deferred lighting pass (GGX + Smith + Fresnel-Schlick), for the directional
and point lights, with a flat ambient term. No image-based lighting.
**Builds on:** v0.7 MRT G-buffer + deferred lighting (merged to `main`).

## 1. Goal & Motivation

Through v0.7 the deferred lighting pass uses Lambert diffuse only — no specular, no
material response. v0.8 makes lighting **physically based**:

1. **Materials** gain `metallic` and `roughness` scalars.
2. The **lighting pass** swaps Lambert for the Cook-Torrance microfacet BRDF
   (GGX normal distribution, Smith geometry, Fresnel-Schlick), with
   energy-conserving diffuse and `F0 = mix(0.04, albedo, metallic)`.
3. **Scene materials** vary (polished metal, brushed metal, glossy plastic, rough
   ground) so the coloured point lights produce specular highlights that read as
   distinct surfaces.

The G-buffer already carries albedo, world normal, and world position; the two new
scalars ride in its currently-unused alpha channels, so no new attachment is
needed. Real ambient reflections (IBL) are a later milestone; v0.8 uses a flat
ambient.

## 2. Roadmap context (this spec is v0.8 only)

| Milestone | Adds |
| --- | --- |
| v0.1–v0.7 (done) | Command buffer, sort keys, mesh path, JobSystem, camera UBO, HDR + tonemap, render passes + RTT, shadow map, MRT G-buffer + deferred lighting. |
| **v0.8 (this spec)** | `metallic`/`roughness` materials, Cook-Torrance BRDF (direct lights) in the lighting pass, packed G-buffer, varied scene materials. |
| Later | Image-based lighting (irradiance + prefiltered env + BRDF LUT), texture maps (albedo/normal/metallic-roughness), AO/emissive G-buffer channels, light culling, bloom/SSAO. |

## 3. Decisions (locked)

| Decision | Choice |
| --- | --- |
| Material | `Material { glm::vec4 baseColor; float metallic; float roughness; ShaderHandle shader; }`; defaults `metallic 0`, `roughness 0.5` |
| G-buffer | No new target: `gAlbedo.a = metallic` (RGBA8), `gNormal.a = roughness` (RGBA16F); `gWorldPos.a` stays 1 |
| BRDF | Cook-Torrance: `D` GGX/Trowbridge-Reitz, `G` Smith (Schlick-GGX, direct-light `k=(r+1)²/8`), `F` Fresnel-Schlick; `F0 = mix(vec3(0.04), albedo, metallic)` |
| Diffuse | `kD = (1 - F) * (1 - metallic)`; `kD * albedo / PI` |
| Directional | `radiance = vec3(1.0) * kDirIntensity (3.0)`, times `(1 - shadowPCF)` |
| Point lights | `radiance = color.rgb * color.a (intensity) * attenuation`; `att = clamp(1 - d/radius, 0, 1)²` (unchanged) |
| Ambient | Flat `0.03 * albedo` (no IBL) |
| View vector | Lighting pass reads `u_cameraPos` from the Camera UBO (binding 0); `V = normalize(cameraPos - worldPos)` |
| Scene | Varied materials: metal + brushed metal + plastic swarm, medium-rough walls, rough ground |
| DoD | Swarm shows PBR-differentiated surfaces (sharp metal specular, soft plastic, matte ground) under the shadowed directional + coloured point lights |

## 4. Architecture & data flow

The pass structure (shadow → geometry → lighting → composite) is unchanged; the
change is what the geometry pass writes and how the lighting pass shades.

```
Material { baseColor, metallic, roughness, shader }
  │  executeGeometry: u_baseColor, u_metallic, u_roughness -> mesh.glsl
  ▼
Geometry pass -> G-buffer MRT
  gAlbedo   = vec4(baseColor.rgb, metallic)
  gNormal   = vec4(normalize(worldNormal), roughness)
  gWorldPos = vec4(worldPos, 1)
  ▼
Lighting pass (fullscreen)   [reads Camera UBO binding 0 for cameraPos]
  albedo/metallic <- gAlbedo ; N/roughness <- gNormal ; worldPos <- gWorldPos
  V = normalize(cameraPos - worldPos) ; F0 = mix(0.04, albedo, metallic)
  Lo = ambient(0.03*albedo)
     + CookTorrance(dir)  * (1 - shadowPCF) * dirRadiance
     + Σ CookTorrance(point_i) * pointRadiance_i
  fragColor = vec4(Lo, 1)   (HDR)
  ▼
Composite -> tonemap (Reinhard + gamma), unchanged
```

## 5. Materials

`engine/renderer/ResourceRegistry.hpp`:

```cpp
struct Material {
  glm::vec4 baseColor{1.0f};
  float metallic = 0.0f;
  float roughness = 0.5f;
  ShaderHandle shader = 0;
};
```

Existing aggregate-init callers in the game move from `{{r,g,b,a}, s}` to
`{{r,g,b,a}, metallic, roughness, s}`. Tests that build `Material` by member
assignment are unaffected (new fields default). `registerMaterial` /
`material(handle)` are unchanged in signature and store the two new scalars.

`OpenGLBackend::executeGeometry` sets the extra uniforms per material:

```cpp
shader->setFloat4("u_baseColor", mat.baseColor);
shader->setFloat("u_metallic", mat.metallic);
shader->setFloat("u_roughness", mat.roughness);
```

## 6. Geometry shader (`mesh.glsl`)

Add the two uniforms and pack them into the G-buffer alphas:

```glsl
// fragment
uniform vec4 u_baseColor;
uniform float u_metallic;
uniform float u_roughness;
layout(location = 0) out vec4 gAlbedo;
layout(location = 1) out vec4 gNormal;
layout(location = 2) out vec4 gWorldPos;
void main() {
  gAlbedo = vec4(u_baseColor.rgb, u_metallic);
  gNormal = vec4(normalize(v_worldNormal), u_roughness);
  gWorldPos = vec4(v_worldPos, 1.0);
}
```

The vertex shader is unchanged from v0.7.

## 7. Lighting pass (Cook-Torrance)

The lighting shader (backend-owned, inline) adds the Camera UBO block (binding 0)
and the BRDF, replacing the Lambert accumulation. Helper functions:

```glsl
const float PI = 3.14159265359;

float distributionGGX(vec3 N, vec3 H, float rough) {
  float a = rough * rough;
  float a2 = a * a;
  float ndh = max(dot(N, H), 0.0);
  float d = ndh * ndh * (a2 - 1.0) + 1.0;
  return a2 / (PI * d * d);
}
float geometrySchlickGGX(float ndv, float rough) {
  float r = rough + 1.0;
  float k = (r * r) / 8.0;
  return ndv / (ndv * (1.0 - k) + k);
}
float geometrySmith(vec3 N, vec3 V, vec3 L, float rough) {
  return geometrySchlickGGX(max(dot(N, V), 0.0), rough) *
         geometrySchlickGGX(max(dot(N, L), 0.0), rough);
}
vec3 fresnelSchlick(float cosT, vec3 F0) {
  return F0 + (1.0 - F0) * pow(clamp(1.0 - cosT, 0.0, 1.0), 5.0);
}
```

Per-light term (`radiance` = the light's colour × intensity × attenuation;
directional attenuation = 1 and its result is scaled by `(1 - shadow)`):

```glsl
vec3 brdf(vec3 N, vec3 V, vec3 L, vec3 albedo, float metallic, float rough,
          vec3 F0, vec3 radiance) {
  vec3 H = normalize(V + L);
  float ndl = max(dot(N, L), 0.0);
  float ndv = max(dot(N, V), 0.0);
  float D = distributionGGX(N, H, rough);
  float G = geometrySmith(N, V, L, rough);
  vec3 F = fresnelSchlick(max(dot(H, V), 0.0), F0);
  vec3 spec = (D * G * F) / (4.0 * ndv * ndl + 0.0001);
  vec3 kd = (vec3(1.0) - F) * (1.0 - metallic);
  return (kd * albedo / PI + spec) * radiance * ndl;
}
```

`main`: unpack `albedo`/`metallic` from `gAlbedo`, `N`/`roughness` from `gNormal`,
`worldPos` from `gWorldPos` (keep the background guard on `dot(N,N) < 0.5`);
`V = normalize(u_cameraPos.xyz - worldPos)`; `F0 = mix(vec3(0.04), albedo,
metallic)`; accumulate `0.03 * albedo` ambient + directional (× `1 - shadowPCF`) +
each point light; write HDR. `shadowPCF` and the point-light loop structure carry
over from v0.7. The directional light colour is a constant `vec3(1.0) * 3.0`.

## 8. Scene materials (game)

`BotArenaGame` registers PBR materials so the range is visible:

| Material | baseColor | metallic | roughness |
| --- | --- | --- | --- |
| Ground | `(0.35,0.35,0.38)` | 0.0 | 0.85 |
| Walls | `(0.7,0.7,0.7)` | 0.0 | 0.5 |
| Swarm 0 (polished metal) | `(1.0,0.78,0.34)` gold | 1.0 | 0.25 |
| Swarm 1 (brushed metal) | `(0.6,0.7,0.9)` steel-blue | 1.0 | 0.45 |
| Swarm 2 (glossy plastic) | `(0.85,0.28,0.24)` red | 0.0 | 0.30 |

Nothing else in the game changes (point lights, camera, swarm layout as v0.7).

## 9. Testing

- **Unit (Catch2, no GL):** extend `tests/test_resource_registry.cpp` — a
  `registerMaterial` with explicit `metallic`/`roughness` round-trips through
  `material(handle)`, and a default-constructed `Material` reports `metallic == 0`
  and `roughness == 0.5`. (The BRDF math lives in-shader and is not duplicated
  CPU-side.)
- **Behavioral:** `BOTARENA_SCREENSHOT` shows the swarm with PBR-differentiated
  surfaces — tight bright specular glints on the metal cubes, broader soft
  highlights on the plastic, a matte ground — under the shadowed directional light
  and coloured point lights, tonemapped.

## 10. Out of scope (later milestones)

Image-based lighting (irradiance map + prefiltered environment + BRDF LUT) for
real ambient reflections; texture maps (albedo/normal/metallic-roughness/AO)
instead of flat per-material scalars; AO and emissive G-buffer channels;
multi-scatter energy compensation; light culling; bloom/SSAO. The `Material`
fields and the analytic BRDF are the foundation these extend.
